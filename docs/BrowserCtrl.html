BrowserCtrl

 

class BrowserCtrl : public WithFileSelLayout<ParentCtrl> 

 

 

A configurable and embeddable file selector widget similar to FileSel. This class provides a callback based uniform interface for building file system transparent file selectors and browsers.

BrowserCtrl& StdMenu(bool b = true)

Toggles the default context menu. The default context menu consists of the standard file browser actions (MkDir, Rename, Delete, Browse, Select etc.). Returns *this for method chaining. Note that standard menu is not mandatory and is independent of the user menu which can be constructed using the WhenBar callback. However, when the standard menu is enabled the user menu will be automatically appended to it

.

Event<Bar&> WhenBar

This event is triggered upon the right mouse button click on file list. Allows adding a user-menu to the browser. Note that when the standard menu is enabled user-supplied menu will be appended to it.

 

 

Event<> WhenSel

This event is triggered when an item or multiple items are selected.

 

 

Gate<> WhenDirUp

This event is triggered when a dirup request is dispatched.

 

 

Gate<> WhenLoad

This event is triggered when the browser attempts to navigate (load) a directory.

 

 

Event<> WhenDrag

This event is triggered when an item is dragged from the file list. Requires drag and drop to be enabled.

 

 

Event<PasteClip> WhenDrop

This event is triggered whenever items are dropped onto the file list. It has the PasteClip as its parameter.

 

 

Event<> WhenDropFiles

This event is a specialization of WhenDrop. It is triggered whenever the dropped items are file(s)

 

 

Gate<const String&> WhenDelete

This event is triggered when an item deletion request is dispatched. It has the path to delete as its parameter.

 

 

Gate<const String&> WhenMkDir

This event is triggered when a new directory is being created. It has the path of the new directory to be created as its parameter.

 

 

Gate<const String&, const String&> WhenRename

This event is triggered when directory entry is to be renamed. It has the paths to the old name and the new name of the item to be renamed as its parameters.
